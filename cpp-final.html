00:00 — Введение

В начале доклада спикер объясняет, что знакомые темы C++ могут иметь много скрытых деталей. Даже если разработчик знает, как написать класс и наследование, ему полезно понимать, что происходит внутри программы.

08:00 — Наследование в C++

Наследование позволяет создавать один класс на основе другого. В простом случае объект наследника содержит часть базового класса и добавляет свои поля и методы.

Простыми словами:

Если есть класс Animal, то класс Dog может унаследовать его общие свойства и добавить свои особенности.

18:00 — Виртуальные функции

Виртуальные функции нужны, чтобы метод выбирался во время выполнения программы. Это удобно, когда мы работаем через указатель или ссылку на базовый класс, но фактически объект является наследником.

Без виртуальной функции может вызваться метод базового класса. С виртуальной функцией C++ сможет выбрать правильный метод наследника.

30:00 — V-таблица

V-таблица — это служебный механизм, который помогает программе понять, какую виртуальную функцию нужно вызвать. Обычно программист напрямую с ней не работает, но важно знать, что она может влиять на размер объекта и скорость работы.

Объект

Хранит поля класса. Например, возраст животного или вес собаки.

vptr

Служебный указатель, который может появляться у объектов с виртуальными функциями.

vtable

Таблица, через которую программа находит нужную виртуальную функцию.

45:00 — Основные выводы

  • Наследование в C++ — это не только удобный синтаксис.
  • Виртуальные функции добавляют гибкость, но имеют свою цену.
  • V-таблицы помогают реализовать полиморфизм.
  • Нужно понимать внутреннее устройство C++, чтобы писать более осознанно.

Примеры кода

В докладе объясняется, что наследование и виртуальные функции влияют не только на синтаксис программы, но и на то, как объект устроен в памяти. Поэтому ниже приведены несколько примеров, которые помогают понять тему.

Пример 1. Обычное наследование

Здесь класс Dog наследуется от класса Animal. Объект Dog содержит часть Animal и добавляет своё поле. 

class Animal {
            public:
                int age;
            
                void eat() {
                    // общее действие для всех животных
                }
            };
            
            class Dog : public Animal {
            public:
                int weight;
            
                void bark() {
                    // действие, которое есть только у собаки
                }
            };
            
            int main() {
                Dog dog;
            
                dog.age = 3;
                dog.weight = 15;
            
                dog.eat();
                dog.bark();
            }

В этом примере нет виртуальных функций, поэтому вызовы методов достаточно простые. Компилятор заранее понимает, какую функцию нужно вызвать.

Пример 2. Виртуальный метод

Виртуальная функция нужна для полиморфизма. Это значит, что программа может вызвать метод наследника через указатель на базовый класс. 

#include <iostream>
            
            class Animal {
            public:
                virtual void sound() {
                    std::cout << "Some animal sound" << std::endl;
                }
            };
            
            class Dog : public Animal {
            public:
                void sound() override {
                    std::cout << "Bark" << std::endl;
                }
            };
            
            int main() {
                Dog dog;
                Animal* animal = &dog;
            
                animal->sound();
            }

Хотя переменная имеет тип Animal*, фактически она указывает на объект Dog. Благодаря слову virtual будет вызван метод Dog::sound.

Пример 3. Виртуальный деструктор

Если объект наследника удаляется через указатель на базовый класс, у базового класса должен быть виртуальный деструктор. 

#include <iostream>

            class Animal {
            public:
                virtual ~Animal() {
                    std::cout << "Animal destroyed" << std::endl;
                }
            };

            class Dog : public Animal {
            public:
                ~Dog() override {
                    std::cout << "Dog destroyed" << std::endl;
                }
            };

            int main() {
                Animal* animal = new Dog();

                delete animal;
            }

Если убрать virtual у деструктора базового класса, при удалении объекта может возникнуть ошибка логики: деструктор наследника может не вызваться корректно.

Пример 4. Множественное наследование

Множественное наследование сложнее обычного, потому что объект может содержать сразу несколько базовых частей. 

class Printer {
            public:
                virtual void print() {
                    // печать документа
                }
            };

            class Scanner {
            public:
                virtual void scan() {
                    // сканирование документа
                }
            };

            class MFP : public Printer, public Scanner {
            public:
                void print() override {
                    // печать на МФУ
                }

                void scan() override {
                    // сканирование на МФУ
                }
            };

Такой код может быть удобным, но внутреннее устройство объекта становится сложнее. Именно поэтому в докладе уделяется внимание тому, как классы располагаются в памяти.
